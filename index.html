La gestion de la donnée au cœur du projet

Comment les informations transitent et sont traitées entre l'utilisateur et l'API.

1. Structuration du contexte

Le LLM est alimenté par un "System Prompt" strict contenant les données de la boutique (catalogue, prix en FCFA, horaires). Cela évite les hallucinations et garantit que l'IA reste dans son rôle de conseiller de vente Namplee.

2. Communication Frontend/Backend

Les requêtes asynchrones transitent via des payloads JSON structurés. Le backend Flask assure le parsing, la validation des inputs utilisateurs, et la gestion des erreurs avant d'interroger le modèle.

// Exemple de payload JSON traité

{

"user_session": "sess_8921",

"intent": "check_stock",

"entities": {

"product": "karite_pur",

"quantity": 2

},

"raw_message": "Je veux 2 pots de karité"

}
